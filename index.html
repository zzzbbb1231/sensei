SenseiBA

Open-source, distributed OLAP datastore

Powering various analytics projects inside LinkedIn.

Some Features:

  • Small memory footprint
  • Structured and faceted search
  • BQL: SQL-like query language
  • Fast key-value lookup
  • High performance under concurrent heavy update and query volumes
  • Hadoop integration
  • Sophisticated aggregation and built-in M/R support

SenseiBA the Name

It shares the same name with its parent project SenseiDB.

Sensei (先生) means teacher or professor in Japanese (http://en.wikipedia.org/wiki/Sensei).

It has the same pronunciation and writing as the Chinese word.

What makes it different from other OLAP solutions?

It can store several billions documents in memory on a single machine with 64 RAM. We all know that hitting the disk is expensive. It takes several milliseconds per a disk seek and that kills the performance. That's why we've made a conscious decision to achieve best possible index compression, so that we can do the index scan in RAM even for a huge dataset. At the same time it's based on the search oriented database and is extremely fast. LinkedIn engineers spend several years to achieve stability and best possible performance for the SenseiDB. And SenseiBA inherits all those cool features

BQL

Browse Query Language (BQL) is an SQL-like language used to to interface with Sensei.

Example BQL: 

  select sum(ArrDelay), avg (DepDelay) 
     from Flights 
     where UniqueCarrier = 'US' group by Dest

What SenseiBA isn't?

SenseiBA doesn't support transactions. It is suitable for append only updates and doesn't offer data mutability. In order to achieve index compression and store 10x more docs per node we had to get rid of Lucene, that's why SenseiBA doesn't support full text search and real time indexing. Although the last one can be achieved by taken hybrid approach and deploying SenseiDb and SenseiBA in the same cluster. In our roadmap we have a plan to implement realtime indexing for the project.

Hadoop integration

Ideally indexes are computed externally (in most cases in Hadoop) and pushed to the SenseiBA via the rest interface

SenseiBA offers the ability to bootstrap from data in HDFS via a built-in Map-reduce indexing job. This can be a very appealing for a wide-variety of data-warehousing applications.
